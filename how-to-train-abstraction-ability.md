# 如何锻炼高度抽象能力

抽象能力本质上是**看到不同事物背后共同结构**的能力。

## 按难度递进的路径

### 入门：建立直觉

| 资料 | 为什么推荐 |
|------|-----------|
| 《哥德尔、艾舍尔、巴赫》 | 用音乐、绘画、对话讲递归、自指、形式系统 |
| 《如何解题》(波利亚) | 数学思维方法论，教你"往上抽象一层"看问题 |
| 《程序员的数学》系列 | 用编程视角讲离散数学，接地气 |

### 中级：核心抽象训练

| 资料 | 训练什么 |
|------|----------|
| **抽象代数入门** (如 Pinter 的 *A Book of Abstract Algebra*) | 群、环、域 — 学会看"结构"而不是"数" |
| **Category Theory for Programmers** (Bartosz Milewski) | 范畴论，免费在线，有视频，程序员友好 |
| **SICP** (计算机程序的构造和解释) | 用Lisp教抽象，经典中的经典 |

### 进阶：深度抽象

| 资料 | 说明 |
|------|------|
| **《Conceptual Mathematics》** (Lawvere) | 范畴论入门，强调直觉 |
| **Haskell / language-ext 实践** | 用类型系统强迫你抽象 |
| **《Types and Programming Languages》** | 类型论，理解"类型即命题" |

## 关键练习方法

书只是输入，真正提升靠**刻意练习**：

1. **找共同模式**：看到两段相似代码，问自己"共同结构是什么？能抽出来吗？"

2. **命名抽象**：能给抽象出的东西起个准确的名字（Monoid、Functor、Strategy...）

3. **跨领域类比**：
   - 加法和字符串拼接有什么共同点？→ Monoid
   - Promise 和 Option 有什么共同点？→ Monad
   - 地图和函数有什么共同点？→ Functor

4. **用最少假设解决问题**：不要假设是"数字"，假设它是"可比较的东西"就够了

## 建议学习路径

```
SICP 或 Category Theory for Programmers
         ↓
   抽象代数入门
         ↓
  用 Haskell/language-ext 写真实项目
         ↓
    Types and Programming Languages (可选)
```

## 最重要的一点

抽象能力不是读出来的，是**反复重构代码、追问"本质是什么"练出来的**。
